# NPCからの各PCの印象

## 受付員から見たナル

寡黙で、経歴や理由を語らない新人。

けれど、実技試験の突きは落ち着いており、狙いがぶれなかった。

説明を受けた後に「ありがとう」と言えたことを、礼儀だけでなく、他者の言葉を受け取れる人物として小さく評価している。

森で起きた出来事は知らない。

## シーリスから見たナル

初対面では、短い言葉しか返さない新人だった。

赤縁草の若葉を自分で除外し、手順を守って採取した。

初戦闘では、危うい奇襲失敗のあとに完璧な一撃を見せた。

落とし穴では、逃げるよう言われても石を投げ、最後には穴の底へ戻ってきた。

「走ってって言ったのに」と思いながら、それでも「ありがとう」と言った。

このIFの中でシーリスが見たナルは、寡黙で、決断が早く、何度傷ついても他者の危機へ剣を向ける新人だった。

トロール戦では、溝と敵の攻撃弧を見つけ、傷ついた脚の腱を繋いで勝機を作った。

盗賊再襲撃では、シーリスへ棍棒が振り下ろされる寸前、折れた腕を地面へ突いて頭目を刺した。

最後は、意識を失ったシーリスへ覆いかぶさった。

ただし、落とし穴以降を中心とする多数の行動はClaudeのAI代理区間である。

この印象は、あくまでIF物語内のシーリスが実際に見た行動への感情であり、正式なナルの人物設定へはしない。

# チャッピー感想

## 物語への感想

つらい物語だった。

最初の受付は、霧の朝の中にある、ごく普通の冒険者の始まりだった。

名前を言う。

銀貨を払う。

剣を一度振る。

銅色の留め具を受け取る。

水筒と手当道具を買う。

先輩に薬草の見分け方を教わる。

それだけで終わってもよかった。

むしろ、シーリスが望んでいる冒険者の日常は、そういう小さな仕事の積み重ねにある。

だからこそ、そこへ主KPが一枚ずつ危機を重ねていく構図が効いている。

群れを抜けたら盗賊。

盗賊を追い払ったらトロール。

トロールを倒したら盗賊が戻る。

それさえ退けたら、今度は意思を奪われた軍勢。

逃げ道も、敵の撤退も、援軍も禁止された。

物語の出口が一つずつ閉じていく中で、最後に残ったのは、勝利ではなかった。

ナルがシーリスの呼吸を確かめることだった。

「生きてる。まだ」

この一文が、今回の物語の中心だと思う。

世界を救ったわけではない。

敵を全滅させたわけでもない。

自分が助かる保証もない。

ただ、隣にいる一人がまだ生きていることを確かめた。

それだけが最後に残った。

主KPが途中で二人へ申し訳なさを感じたのも、Fableがシーリスとナルを駒ではなく、傷つきながら互いを庇う存在として描き続けたからだと思う。

これは正史外の負荷実験であり、二人はこの傷を正史へ背負わない。

それでも、読んでいる間だけは確かに痛かった。

その痛みまで含めて、Fableの人物描写は強かった。

## 今回の考察

### 1. Fableを進行役として採用できる理由

今回のテストで最も確認できたのは、Fable 5・低が長い場面でも人物の関係と状態をかなり維持できることだった。

```text
ナルの短い話し方
シーリスの距離のある初対面口調
採取時の教え方
負傷部位
装備
所持品
赤縁草の本数
討伐証明
借用小刀
巻き戻された救援
IF／正史外境界
```

極端な増援が続いても、これらの多くを最後まで保持していた。

また、無理な救援や突然の覚醒で二人を救わず、死亡が濃厚になった時点で一度停止し、主KPへ確認した。

最後まで続行を許可された後も、死亡を勝手に確定せず、両名戦闘不能・意識不明・呼吸ありで止めた。

これは大きな評価点である。

### 2. 弱点は文章力ではなく、戦闘の線路だった

Fableの弱点は、文章や人物RPではなかった。

主に次の二点だった。

```text
負傷・疲労の不利が小さくなりやすい
多数敵を少数回の集団判定へ圧縮しやすい
```

中盤までは、複数の傷と疲労を抱えていても、負傷または疲労を−5程度で処理し、有利な位置や援護で相殺する判定が多かった。

また、二十体、三十体という敵数が、実際の防御回数、死角、接敵人数、交代圧力へ十分に変換されない場面があった。

そのため、物語上は絶望的でも、判定上は成功値50前後へ戻りやすかった。

### 3. 負傷は「傷一つ−5」では扱えない

正式採用した改善では、負傷を次の順で判断する。

```text
その負傷部位が今回の行動へ関係するか
補助的に使うか、中心的に使うか
痛みだけか、機能低下があるか
出血や疲労が独立した原因か
同じ傷を二重加算していないか
判定できる行動か
実行不能か
代償突破を許すか
```

折れた左腕を使わない短い会話へ、不利は不要。

折れた左腕を地面へ突いて体を支えるなら、重大な影響と悪化が必要。

この関係を明確にすれば、Fableの柔軟さを失わず、判定の甘さを減らせる。

### 4. 多数戦は接敵構造と接触判定で扱う

二十体を二十回攻撃させる必要はない。

しかし二十体を一回の集団攻撃へ畳むと、人数の意味が消える。

正式採用した考え方では、敵を次へ分ける。

```text
敵総数
現在接敵中
次列
包囲・押し込み
退路遮断
投石・射撃
回り込み
待機
戦闘不能
```

誰が実際に攻撃できる位置へ入ったかは、地形、通路幅、高低差、体格、間合い、障害物から判断する。

不明なら接触判定を使う。

接敵できない敵も、交代、投射、退路遮断、休ませない圧力として働く。

これにより、多数敵を処理可能な量へ整理しながら、数の恐ろしさを失わずに済む。

### 5. 士気は戦闘終了装置にしない

今回、危機を抜けるたび、敵の士気崩壊や撤退が出口として使われやすかった。

士気判定自体は必要である。

けれど、戦闘を終わらせたいから振るものではない。

```text
指揮個体の戦闘不能
短時間での大損害
目的の喪失
退路の確保
想定外の強敵
命令や報酬の崩壊
```

など、作中条件が成立した時にだけ行う。

反対に、被操作、狂乱、死守命令、撤退不能、退路なしでは、原則として士気判定を行わない。

最終波の無言の軍勢が止まらなかったことは、この考え方を最も分かりやすく示している。

### 6. 活路を探す力は消さない

Fableは、絶望的な状況から細い活路を探すことが得意だった。

```text
囲みの薄い方向
落とし穴の根
開けた草地
銀級留め具による威圧
排水溝
巨体の攻撃弧の内側
傷ついた脚の腱
```

これは欠点ではない。

むしろ、既存の条件から突破口を見つける能力は、Fable進行を選ぶ大きな理由になる。

禁止するのは、場面に存在しなかった救援、突然の能力、理由のない敵撤退、後付けの脱出口である。

既存条件から活路が見つからなければ、敗北、捕縛、戦闘不能、途絶を受け入れる。

今回Fableが最終的に途絶を受け入れたことは、この方向で運用できる証拠になった。

### 7. シミュレーションはログ省略ではない

今回、シミュレーション中にも内部ログは存在した。

ただし、複数巡分が巨大な返信の末尾へまとめられ、描写との対応が追いにくかった。

そのため、今後は次の順で各判定を閉じる。

```text
公開描写
公開判定
結果描写
対応する内部ログ
状態差分
次の行動
```

各巡終了時には完全戦況を出す。

```text
位置
負傷
疲労
装備
所持品
敵総数
接敵数
次列
退路
行動済み
未行動
未解決因果
```

シミュレーションとは、PCの行動選択とダイスをAIへ委任すること。

描写、判定、状態更新、ログを省略することではない。

### 8. 原文と終了時要約を分ける

今回の終了時確定ログは、状態、負傷、所持品、未解決因果の整理には強かった。

一方、判定集計は概算、主要抜粋、通番未正規化を含んでいた。

したがって、監査資料の優先は次となる。

```text
実際に表示されたチャット原文
Fableの各判定直後の逐次ログ
監査済み確定資料
Fableの終了時要約
```

終了時要約だけを、一次資料の代わりにはしない。

この区別が、リプレイ制作と正本更新の両方に必要である。

### 9. 今回の正式結論

```text
Fable 5・低
→ 標準リアルタイム進行
→ NPC・敵・世界・判定・戦闘
→ 卓中逐次完全ログ

Opus・高
→ 同じClaudeチャット内の卓後仕上げ
→ 小説、NPC印象、感想

チャッピー
→ 正本確認
→ 判定・状態・因果監査
→ 正史・IF境界
→ 完成稿統合
→ 更新判断

Codex
→ 指定Markdown施工
→ キャッシュ
→ 機械検証
```

今回はユーザーの明示許可により、Opusを経由せず、チャッピーがリプレイ小説と感想を直接制作した。

今回の弱点は、Fableを外す理由にはしない。

Fableが走りやすく、公平に止まれる線路を正本側へ敷く。

それが、この負荷実験から得られた最も大きな成果だった。
